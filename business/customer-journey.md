 # Customer-Journey.md

## Aware

### Trigger Event
- Developer encounters a need for a TypeScript-specific refactoring tool while working on a project.
- Developer searches for solutions online and discovers `type-refactor`.

### Friction Points
- Lack of awareness about the tool's existence.
- Confusion due to the variety of refactoring tools available.

### User Emotions
- Frustration with current refactoring methods.
- Curiosity about a tool designed specifically for TypeScript.

### Opportunities to Delight
- Clear and concise marketing materials explaining the benefits of `type-refactor`.
- Testimonials from satisfied users.

### Metric
- Number of unique visitors to the `type-refactor` website.

## Consider

### Trigger Event
- Developer visits the `type-refactor` website or GitHub repository.

### Friction Points
- Lack of detailed documentation or tutorials.
- Uncertainty about the tool's compatibility with the developer's project setup.

### User Emotions
- Interest in the tool's features and capabilities.
- Skepticism about the tool's effectiveness.

### Opportunities to Delight
- Easy-to-understand documentation and tutorials.
- A free trial or demo version of the tool.

### Metric
- Number of page views on the `type-refactor` website.
- Number of downloads of the trial or demo version.

## Try

### Trigger Event
- Developer decides to try `type-refactor`.

### Friction Points
- Complex installation process.
- Learning curve associated with using the tool.

### User Emotions
- Excitement about the potential benefits of the tool.
- Anxiety about the learning curve and installation process.

### Opportunities to Delight
- A user-friendly interface.
- Excellent customer support during the trial period.

### Metric
- Number of successful installations.
- Number of support tickets opened during the trial period.

## Adopt

### Trigger Event
- Developer decides to adopt `type-refactor` for their project.

### Friction Points
- Integration issues with existing project setup.
- Ongoing learning curve as the developer becomes more proficient with the tool.

### User Emotions
- Relief at finding a solution to their refactoring needs.
- Satisfaction with the tool's effectiveness.

### Opportunities to Delight
- Regular updates and improvements to the tool.
- A community of users who can share tips and best practices.

### Metric
- Number of active users.
- Number of positive reviews or testimonials.

## Expand

### Trigger Event
- Developer becomes a power user of `type-refactor` and is interested in expanding their use of the tool.

### Friction Points
- Limited features in the free version.
- Cost of upgrading to a premium version.

### User Emotions
- Loyalty to the tool due to its effectiveness.
- Willingness to invest in the tool to unlock additional features.

### Opportunities to Delight
- A tiered pricing structure that offers a range of features at different price points.
- A referral program that rewards users for bringing in new customers.

### Metric
- Number of premium users.
- Number of referrals made by existing users.
